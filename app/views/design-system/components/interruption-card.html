Interuption card

A component that interrupts the user flow with important information and must be acknowledged before continuing.

Example

  • Example
  • Code
  • Alert

When to use this component

  • to draw a users attention to important information specific to that point in their journey
  • allow users to acknowledge something before continuing
  • If important information is styled as normal page content and users are missing it, this is unexpected so helps bring attention to important or essential information

When not to use this component

  • for large amounts of content
  • the effectiveness of this pattern is based on it's unexpected appearance, so should be used sparingly

Consider how content can be made clearer or easier to understand by users before using this component in services.

The interruption card should:

  • have a large heading govuk-heading-l
  • use larger body text govuk-body-l
  • use a "Continue" button to progress to the next step in the users journey
