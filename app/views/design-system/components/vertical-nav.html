Vertical navigation

Use the vertical navigation to allow users to access different sections or pages in relation the current page. Useful for menus and application navigation.

The menu is an adaptation of the vertical navigation menu on the GOV.UK Design System.

Basic implementation

Example

  • Code
  • Domain names 
<nav class="gc-vertical-nav">
    <ul class="gc-vertical-nav__section">
        <li class="gc-vertical-nav__section-item {{alert-c-nav-active}}">
            <a class="gc-vertical-nav__link " href="#">Alert</a>
        </li>
    </ul>
</nav>

Implementation with counts

Selected navigation item

Styling on the navigation item to show the user the item is the currently selected menu item.

This is achieved by applying the class gc-vertical-nav__section-item--current to the relevant menu item

When to use this component

Use this component to structure web applications with a vertical navigation.

This component should be placed within a app-pane__subnav or govuk-grid-column-one-third container.

The menu should be in scope to the information it is representing. For example, in the public register, the menu is used to show domain names, trading names, sanctions and settlements for a selected licensed business.

When not to use this component

Understand first what content is being displayed before considering using this component. It should not be used in standard websites where a contents list might be better used.

How it works

Built on the standard ul and li structure so it can be easily navigated and understood by screen readers and keyboard-only users.

Research on this component

This component was developed, iterated and tested during the public register project. Users were very receptive to the menu and it aided navigation of different information in relation to register entries.

It is also used within this service manual.
