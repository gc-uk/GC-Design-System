Notifications

Brings important information to the user's attention.

This pattern is used instead of the GOV.UK Design System panel component (link opens in a new window)

It is derived from the GOV.UK Design System error component (link opens in a new window)

Variations of this component

There are 2 variations of this component.

Contents

  • Example
  • Code

Information

Used when the user needs to be made aware of something they need to do. 

<div class="gc-notification gc-notification--info">
    <h2 class="govuk-heading-m">Information</h2>
    <p class="govuk-body">Used when the user needs to be made aware of something they need to do.</p>
</div>

Success

Used when the user needs to be told that they have completed something. 

<div class="gc-notification gc-notification--success">
    <h2 class="govuk-heading-m">Information</h2>
    <p class="govuk-body">Used when the user needs to be made aware of something they need to do.</p>

When to use this component

This component is used when a user needs to be made aware of something they need to complete or that something has happened. For example:

  • a maintenance is due
  • at the end of a transaction to confirm completion

Use standard typography rules with this component, this includes heading and body classes.

Use buttons where required to trigger an action to be done.

    <h2 class="govuk-heading-m">This is due</h2>
    <p class="govuk-body">You need to do this action by 15 November 2019.</p>
    <button type="button" class="govuk-button">Start now</button>
</div>

Success guidance

When using the success style, all relevant information for the transaction must be contained within the notification. For example:

  • the outcome
  • a reference number (if applicable)
  • what has been sent to the user (if applicable)

Do not include next steps in the notification component. For detailed guidance of how to use this when a transaction is complete, see the transaction complete pattern.

When not to use this component

Do not use this component to:

  • alert the user to an error - use the error summary component
  • give the user general information, use body content
  • interrupt the user journey, use the interruption component

Services using this component

  • Manage your personal licence
  • User research registration service
  • Apply for a licence

Research on this component

This info style has been tested with users of the manage your personal licence service. It has tested well and had the desired effect for drawing the user's attention to their maintenance needing to be completed.

The success style has been tested through A/B testing with the GOV.UK Design System panel component and users preferred our Notification component as it was consistent with our other notification styles.

The style and user research for this component have been fed back to the GOV.UK design team in relation to the panel component.
