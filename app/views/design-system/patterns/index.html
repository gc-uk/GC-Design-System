Patterns

Patters are predefined sections which implement one or more components and ensure consistency across services.

A pattern must be implemented in a standard way across all services. This reduces the issues users may face when using multiple services.

This is a key UX law where users recognise patterns over recalling patterns. A user is more likely to recognise a pattern for when something is complete, over recalling what was being displayed.
